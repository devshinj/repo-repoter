# 📋 Repo Reporter 사용자 가이드

> Git 커밋을 자동 수집하고 AI로 분석하여 일일 업무 보고서를 만들어주는 서비스입니다.

---

## 📌 서비스 소개

**Repo Reporter**는 여러분이 매일 작성하는 Git 커밋을 자동으로 수집하고, Gemini AI가 분석하여 **업무 보고서를 자동 생성**해주는 서비스입니다.

### 이런 분들에게 추천합니다

- 매일 업무 보고서를 수동으로 작성하는 개발자
- 여러 저장소에 걸쳐 작업하여 기록 정리가 번거로운 분
- 팀원의 작업 내역을 한눈에 파악하고 싶은 팀 리드

### 핵심 기능 요약

| 기능 | 설명 |
|------|------|
| 🔄 자동 커밋 수집 | 등록한 저장소에서 15분마다 새 커밋을 자동 수집합니다 |
| 🤖 AI 업무 분석 | Gemini AI가 커밋을 분석하여 업무 단위로 정리합니다 |
| 📊 캘린더 뷰 | 날짜별 커밋 히트맵으로 작업량을 시각적으로 확인합니다 |
| 📝 보고서 생성 | 날짜/기간별 업무 보고서를 원클릭으로 생성합니다 |
| 🔐 다중 저장소 | GitHub, GitLab, Gitea 등 모든 Git 호스팅을 지원합니다 |

---

## 🚀 시작하기

### 1. 회원가입

서비스에 처음 접속하면 회원가입이 필요합니다.

**경로:** `/register`

1. **이름**, **이메일**, **비밀번호**를 입력합니다.
2. **회원가입** 버튼을 클릭합니다.
3. 가입이 완료되면 자동으로 로그인됩니다.

---

### 2. 로그인

두 가지 로그인 방식을 지원합니다.

**경로:** `/login`

#### 이메일/비밀번호 로그인

1. 가입 시 사용한 **이메일**과 **비밀번호**를 입력합니다.
2. **로그인** 버튼을 클릭합니다.

#### HRMS 로그인 (사내 시스템)

1. **HRMS로 로그인** 버튼을 클릭합니다.
2. 사내 HRMS 시스템에서 인증을 완료합니다.
3. 최초 로그인 시 계정이 자동으로 생성됩니다.

## ⚙️ 초기 설정

> 💡 **저장소 등록 전에 반드시 Git 토큰을 먼저 등록해야 합니다.**

### 3. Git 토큰 등록

Git 호스팅 서비스(GitHub, GitLab 등)의 Personal Access Token(PAT)을 등록합니다. 이 토큰으로 비공개 저장소의 커밋을 수집합니다.

**경로:** 사이드바 > **설정**

#### GitHub 토큰 발급 방법

1. GitHub에 로그인합니다.
2. **Settings** > **Developer settings** > **Personal access tokens** > **Tokens (classic)** 으로 이동합니다.
3. **Generate new token (classic)** 을 클릭합니다.
4. 다음 권한(scope)을 선택합니다:
   - `repo` — 비공개 저장소 접근
5. 토큰을 생성하고 복사합니다.

#### 토큰 등록 절차

1. **설정** 페이지에서 **새 자격증명 추가** 버튼을 클릭합니다.
2. **서비스 유형**을 선택합니다. (GitHub, Gitea, GitLab, Bitbucket)
3. **라벨**에 구분할 수 있는 이름을 입력합니다. (예: "회사 GitHub PAT")
4. **토큰**에 복사한 PAT를 붙여넣습니다.
5. **등록** 버튼을 클릭합니다.

#### 토큰 관리

| 기능 | 설명 |
|------|------|
| 토큰 마스킹 | 등록된 토큰은 `ghp_xxx...xxx` 형태로 마스킹되어 표시됩니다 |
| 라벨 수정 | 라벨 옆 수정 아이콘을 클릭하면 인라인 편집이 가능합니다 |
| 토큰 갱신 | 토큰이 만료된 경우 새 토큰으로 교체할 수 있습니다 |
| 삭제 | 더 이상 사용하지 않는 토큰을 삭제합니다 |

> ⚠️ **토큰은 AES 암호화로 안전하게 저장됩니다.** 하지만 토큰을 발급할 때 최소 권한만 부여하는 것을 권장합니다.

---

### 4. 저장소 등록

커밋을 수집할 Git 저장소를 등록합니다. 두 가지 방법을 지원합니다.

**경로:** 사이드바 > **저장소 관리**

#### 방법 A: 저장소 선택 등록 (권장)

등록된 토큰을 기반으로 원격 저장소 목록에서 선택합니다.

1. 상단의 **저장소 추가** 버튼을 클릭합니다.
2. **저장소 선택** 탭에서 등록된 자격증명을 선택합니다.
3. 저장소 목록이 자동으로 로드됩니다.
4. 등록할 저장소를 선택하고 **등록** 버튼을 클릭합니다.

#### 방법 B: URL 직접 입력

1. **저장소 추가** 버튼을 클릭합니다.
2. **직접 입력** 탭을 선택합니다.
3. 저장소의 **HTTPS 클론 URL**을 입력합니다.
   - 예: `https://github.com/your-org/your-repo.git`
4. 추적할 **브랜치**를 입력합니다. (기본값: `main`)
5. 사용할 **자격증명**을 선택합니다.
6. **등록** 버튼을 클릭합니다.

> 💡 **여러 저장소를 등록할 수 있습니다.** 등록 후 백그라운드에서 저장소를 자동으로 clone합니다.

---

### 5. Author 태그 설정

저장소에서 **내 커밋만** 수집하려면 Author 태그를 설정합니다.

**경로:** 사이드바 > **저장소 관리** > 각 저장소 카드

1. 저장소 카드에서 **Author 설정** 영역을 찾습니다.
2. Git 커밋에 사용하는 **이름** 또는 **이메일**을 입력합니다.
   - 예: `홍길동` 또는 `gildong@company.com`
3. 입력 후 저장합니다.

> 📌 Author 태그를 설정하지 않으면 해당 저장소의 **모든 커밋**이 수집됩니다.

## 📊 주요 기능

### 6. 대시보드

서비스의 메인 화면으로, 전체 현황을 한눈에 파악할 수 있습니다.

**경로:** 사이드바 > **대시보드** (또는 `/`)

#### 구성 요소

| 영역 | 설명 |
|------|------|
| 📈 통계 카드 | 등록된 저장소 수, 스케줄러 상태, 마지막 동기화 시간 |
| 🟩 기여 히트맵 | 최근 6개월간 날짜별 커밋 수를 GitHub 스타일 히트맵으로 시각화 |
| 📋 저장소 목록 | 저장소별 마지막 커밋 메시지, 언어 배지, 동기화 버튼 |

#### 수동 동기화

자동 동기화(15분 간격)를 기다리지 않고 즉시 커밋을 수집하고 싶다면:

1. 저장소 카드의 **동기화** 버튼을 클릭합니다.
2. 동기화가 진행되는 동안 로딩 표시가 나타납니다.
3. 완료되면 최신 커밋 정보가 업데이트됩니다.

---

### 7. 저장소 관리

등록된 저장소의 상세 정보와 커밋 이력을 확인합니다.

**경로:** 사이드바 > **저장소 관리**

#### 저장소 목록

- 등록된 저장소가 카드 형태로 표시됩니다.
- 각 카드에는 저장소명, 주 언어 배지, 마지막 동기화 시간이 표시됩니다.
- **삭제** 버튼으로 저장소 등록을 해제할 수 있습니다.

#### 저장소 상세

저장소 카드를 클릭하면 상세 페이지로 이동합니다.

1. 상단의 **브랜치 선택** 드롭다운으로 브랜치를 변경합니다.
2. 해당 브랜치의 **최근 커밋 목록**이 테이블로 표시됩니다.
3. 커밋을 클릭하면 **변경된 파일 목록**을 펼쳐볼 수 있습니다.

---

### 8. 태스크 캘린더

날짜별 커밋 현황을 캘린더 형태로 확인하고, 보고서를 생성합니다.

**경로:** 사이드바 > **태스크 캘린더**

#### 캘린더 뷰

- **1개월 / 3개월** 뷰를 전환할 수 있습니다.
- 각 날짜에 커밋 수가 색상 농도로 표시됩니다. (진할수록 커밋이 많음)
- 커밋이 있는 날짜를 클릭하면 우측 패널에 상세 정보가 나타납니다.

#### 저장소 필터

캘린더 상단의 체크박스로 특정 저장소만 필터링할 수 있습니다.

- 여러 저장소를 등록한 경우, 보고 싶은 저장소만 선택합니다.
- 필터를 변경하면 캘린더 히트맵이 즉시 업데이트됩니다.

#### 날짜 클릭 — 일일 커밋 상세

1. 캘린더에서 **특정 날짜**를 클릭합니다.
2. 우측 패널에 해당 날짜의 커밋이 **저장소별 > 브랜치별**로 정리되어 표시됩니다.
3. 각 커밋의 메시지, 작성자, 시간을 확인할 수 있습니다.

#### 기간 선택 — 범위 커밋 요약

1. 캘린더에서 **시작일을 클릭 후 종료일을 클릭**하여 기간을 선택합니다.
2. 또는 상단의 **기간 프리셋**(이번 주 / 이번 달)을 사용합니다.
3. 우측 패널에 선택 기간의 커밋 요약이 표시됩니다.

#### 보고서 생성

1. 날짜 또는 기간을 선택한 상태에서 **보고서 작성** 버튼을 클릭합니다.
2. AI가 커밋을 분석하여 업무 보고서를 자동 생성합니다.
   - **단일 날짜**: 즉시 생성 (몇 초 소요)
   - **기간 범위**: 백그라운드에서 생성 (데이터 양에 따라 시간 소요)
3. 생성 완료 후 보고서 페이지로 이동합니다.

---

### 9. 업무 보고서

AI가 생성한 업무 보고서를 열람하고 편집합니다.

**경로:** 사이드바 > **업무 보고서**

#### 보고서 목록

- 보고서가 **프로젝트(저장소)별**로 그룹화되어 표시됩니다.
- 각 보고서에는 날짜, 유형(단일 날짜/기간)이 표시됩니다.

#### 보고서 상세

보고서를 클릭하면 상세 페이지로 이동합니다.

| 기능 | 설명 |
|------|------|
| 👀 보기 | Markdown으로 렌더링된 보고서를 읽습니다 |
| ✏️ 제목 편집 | 보고서 제목을 클릭하여 수정합니다 |
| 📝 내용 편집 | **편집** 버튼으로 Markdown 원문을 수정합니다 |
| 📋 클립보드 복사 | **복사** 버튼으로 보고서 전체를 클립보드에 복사합니다 |
| 🗑️ 삭제 | 보고서를 삭제합니다 (복구 불가) |

#### 보고서 활용 팁

- 생성된 보고서를 복사하여 **사내 업무 보고 시스템**에 붙여넣기할 수 있습니다.
- AI가 생성한 내용이 부정확한 경우 직접 편집하여 수정할 수 있습니다.
- 기간 보고서는 주간/월간 업무 정리에 유용합니다.

## ❓ 자주 묻는 질문

<details>
<summary><strong>Q. 동기화가 자동으로 되나요?</strong></summary>

네, 서버가 실행 중이면 **15분 간격**으로 자동 동기화됩니다. 대시보드에서 스케줄러 상태와 마지막 동기화 시간을 확인할 수 있습니다. 급하게 최신 데이터가 필요하면 수동 동기화 버튼을 사용하세요.

</details>

<details>
<summary><strong>Q. 비공개 저장소도 수집할 수 있나요?</strong></summary>

네, 설정 페이지에서 등록한 Git PAT에 해당 저장소의 접근 권한(`repo` scope)이 있으면 비공개 저장소의 커밋도 수집됩니다.

</details>

<details>
<summary><strong>Q. GitHub 외에 다른 Git 호스팅도 사용할 수 있나요?</strong></summary>

네, HTTPS clone이 가능한 모든 Git 호스팅을 지원합니다. GitHub, GitLab, Gitea, Bitbucket 등 해당 서비스의 PAT를 등록하고 저장소를 추가하면 됩니다.

</details>

<details>
<summary><strong>Q. 특정 사람의 커밋만 수집하고 싶어요.</strong></summary>

저장소 관리 페이지에서 **Author 태그**를 설정하세요. Git 커밋에 사용하는 이름 또는 이메일을 입력하면 해당 Author의 커밋만 수집됩니다.

</details>

<details>
<summary><strong>Q. 토큰이 만료되면 어떻게 하나요?</strong></summary>

설정 페이지에서 해당 토큰의 **갱신** 기능을 사용하여 새 토큰으로 교체하세요. 저장소를 다시 등록할 필요는 없습니다.

</details>

<details>
<summary><strong>Q. 보고서 내용이 정확하지 않아요.</strong></summary>

AI가 커밋 메시지를 기반으로 분석하므로, 커밋 메시지가 구체적일수록 보고서 품질이 높아집니다. 생성된 보고서는 상세 페이지에서 직접 편집할 수 있습니다.

</details>

<details>
<summary><strong>Q. 여러 브랜치의 커밋을 수집할 수 있나요?</strong></summary>

저장소 등록 시 지정한 브랜치의 커밋이 기본으로 수집됩니다. 저장소 상세 페이지에서는 다른 브랜치의 커밋도 조회할 수 있습니다.

</details>

## 🗺️ 전체 사용 흐름

```
회원가입/로그인
    ↓
설정 > Git 토큰 등록
    ↓
저장소 관리 > 저장소 추가 + Author 태그 설정
    ↓
[자동] 15분마다 커밋 수집 & AI 분석
    ↓
대시보드에서 전체 현황 확인
    ↓
태스크 캘린더에서 날짜/기간 선택
    ↓
보고서 작성 버튼 클릭 → AI 보고서 생성
    ↓
업무 보고서에서 편집 & 복사 → 사내 시스템에 활용
```
